# 说明与使用方式

TLDR: 这个目录用于承接“长风资料结构化入 Dify / 知识底座”这条线的准备工作；阅读和执行时要区分三层：**当前主路线判断**、**当前执行方案**、**后续升级方向**，不要混读。

## 一、这页的作用
- 给这个目录一个统一入口。
- 先把当前已经收敛的主路线说清。
- 再说明当前阶段怎么执行。
- 最后说明这条线后续可以升级到哪里。

## 二、先分清三层

### 1. 当前主路线判断
这是目前最应该优先服从的层。

当前更稳的理解是：
- 文档解析 / OCR：**阿里工具链或同类稳定方案**
- 通用 Markdown 中间层：**MinerU 或同类方案**
- 分类与字段抽取：**Qwen-Plus**
- 入库与归档判断：**现有模板页 + 规则页**

当前不建议这样理解：
- 不把 **DeepSeek** 当默认主抽取链路中心
- 不把 **Logics-Parsing** 直接当全库默认主路线

这层主要回答的是：**现在先用什么主路线把事情跑稳。**

### 2. 当前执行方案
这是围绕当前主路线展开的落地层。

当前阶段更适合：
- 先跑最小工作流
- 先做样例验证
- 先固化模板、规则、字段映射、去重和证据链
- 先让少量高价值样本稳定落地
- 暂不追求一开始就全目录、全自动、全对象打通

这层主要回答的是：**现在先怎么做。**

### 3. 后续升级方向
这是从当前主路线自然延伸出来的升级层。

后续可以从“Dify 入库准备”继续升级到：
- 通用知识底座
- 多主档结构
- 证据链与生命周期治理
- API / 方案生成 / 参数匹配等多消费端复用

这层主要回答的是：**这条线未来会长成什么。**

## 五、当前阶段最重要的执行原则
- 先服从当前主路线判断，再讨论升级方向。
- 不把远期治理设计直接当成当前样本执行口径。
- 不把“未来可能的完整架构”写成“当前已全部定稿”。
- 不把大蓝图和最小闭环混成一层。
- 当前先以“样例跑通、规则收稳、模板定口径”为优先。

## 六、当前一句话理解
先用 **阿里解析/OCR + MinerU + Qwen-Plus + 模板页/规则页** 跑稳最小闭环；在这个基础上，再逐步升级成可被 Dify、方案生成、参数匹配和 API 共同复用的知识底座。
