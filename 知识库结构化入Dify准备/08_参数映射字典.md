# 参数映射字典

TLDR: 这页用于把不同资料里的同义字段、近义字段、混用写法统一到固定字段口径，避免后续主档、证据、底表和工作流各写各的。

## 一、用途
- 统一产品、厂家、报告、场景、方案等主档的字段命名。
- 统一原始资料、Markdown 清洗稿、抽取结果、Excel 底表之间的字段映射。
- 为后续 Dify、API、方案生成和参数匹配提供同一套字段口径。

## 二、使用原则
- 先定标准字段，再挂常见别名，不反过来。
- 原始写法可以保留在证据层，但主档层应统一写成标准字段。
- 同名字段若语义不同，必须拆开，不能强行并表。
- 价格、税运、标配/选配、当日有效这类强时效字段，优先挂证据层，不直接升格为长期稳定字段。

## 三、产品与设备字段映射
| 标准字段 | 常见别名/原文写法 | 说明 |
| --- | --- | --- |
| 产品名称 | 名称、设备名称、产品名 | 主展示名称 |
| aliases | 别名、简称、旧名称、常见误写 | 用于判重与检索 |
| 型号 | 型号规格、产品型号、规格型号 | 型号主字段 |
| 产品类别 | 类别、设备类型、机型类别 | 用于归类 |
| 制造单位 | 生产厂家、厂家、制造商、企业名称 | 优先挂主体全称 |
| 关键参数 | 技术参数、核心参数、性能参数 | 主档摘要字段 |
| 有效载荷 | 载重、最大载重、任务载荷 | 统一到有效载荷 |
| 续航时间 | 飞行时长、续航、续航能力 | 时间口径统一 |
| 航程 | 作业半径、飞行距离、最大航程 | 注意 km/米单位换算 |
| 防护等级 | 防护级别、IP 等级 | 保留原文等级 |
| 通信方式 | 图传、链路、通信链路 | 与控制方式分开 |
| 适用场景 | 适用行业、应用场景、作业场景 | 场景描述字段 |
| 关联配件 | 挂载、配件、选配件 | 不并入主机参数 |

## 四、检验报告字段映射
| 标准字段 | 常见别名/原文写法 | 说明 |
| --- | --- | --- |
| 报告类型 | 检测报告、检验报告、检定证书、校准证书 | 必须区分具体类型 |
| 报告编号 | 编号、证书编号、报告号 | 判重关键字段 |
| 产品名称 | 样品名称、产品名称 | 尽量回填到主产品对象 |
| 型号 | 型号规格、规格型号 | 与产品主档统一 |
| 制造单位 | 生产单位、制造单位、厂家 | 保持主体一致 |
| 检测机构 | 检验单位、检定机构、检测单位 | 主体字段 |
| 检定依据 | 检验依据、测试依据、执行标准 | 注意与产品执行标准区分 |
| 结论 | 检定结果、检验结论、检测结论 | 保留原文结论 |
| 报告日期 | 签发日期、出具日期 | 日期主字段 |
| 有效期 | 有效至、有效截止日期 | 强时效字段 |

## 八、当前优先统一的歧义项
- `厂家 / 制造单位 / 生产单位 / 企业名称`：默认先统一到“制造单位”或“企业全称”，再看对象层级。
- `检验报告 / 检测报告 / 检定证书 / 校准证书`：必须先分类型，不能只统称“报告”。
- `型号 / 型号规格 / 规格型号`：统一收敛到“型号”。
- `PQ16 / QP16`、`发生器 / 产生器`、`刺穿 / 穿刺`：默认主名称统一，变体进入 aliases。
- `续航 / 飞行时长`、`最大载重 / 有效载荷`：后续参数匹配统一按标准字段落。

## 九、如何使用这页
1. 新样本进入时，先判断对象类型。
2. 再把原文字段对到本页标准字段。
3. 若出现未收录的新写法，补到本页，不直接在主档里临时发明字段。
4. 若同名字段实际语义不同，先在本页拆义，再回写模板。
