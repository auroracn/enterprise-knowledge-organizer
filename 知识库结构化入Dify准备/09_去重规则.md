# 去重规则

TLDR: 这页用于统一判断“哪些资料其实是同一对象、同一原件或同一批次的不同产物”，避免把原件、Markdown、抽取结果、截图和底表误当成多个独立知识对象。

## 一、用途
- 统一主档、证据层、实验产物层的去重判断口径。
- 避免同一产品、同一报告、同一厂家资料被重复建页。
- 为后续批量入库和自动流程提供可复用的判重规则。

## 二、总原则
- 先判断是不是同一对象，再判断保留哪一版。
- 原始文件优先于中间产物，中间产物优先于推断摘要。
- 字段更完整、来源更清楚、可回溯性更强的一版优先。
- 去重不是删除痕迹，而是把同源资料挂回同一主档或证据集合。

## 三、四层去重对象
### 1. 原始文件层
- PDF、Word、Excel、图片、PPT 原件。
- 优先看文件名、路径、日期、来源目录、哈希。

### 2. 中间产物层
- OCR 结果、Markdown 清洗稿、表格抽取结果、字段抽取 JSON。
- 优先挂回原始文件，不单独当正式主档。

### 3. 分类结果层
- 知识库分类输出、目录归类结果、自动命名结果。
- 只作为归并辅助，不替代原始证据。

### 4. 最终确认层
- 人工确认后的产品主档、厂家主档、报告主档、方案主档等。
- 同一对象只保留一个正式主档入口。

## 四、产品主档去重规则
### 判重主键
- 主名称
- aliases
- 型号
- 制造单位
- 原始文件名
- 关联报告编号

### 常见重复情况
- 同一产品同时有说明书、彩页、检测报告、图片、Excel 参数表。
- 同一产品在不同目录中以不同简称出现。
- 同一产品在多轮实验里反复生成 Markdown 或分类结果。

### 处理原则
- 一个产品只保留一个正式主档。
- 说明书、彩页、参数表、报告都挂到同一产品证据集合。
- 命名差异进入 aliases，不另建新对象。

## 五、报告证据去重规则
### 判重主键
- 报告类型
- 报告编号
- 产品名称
- 型号
- 制造单位
- 检测/检定机构

### 常见重复情况
- 同一报告被转成多份 Markdown。
- 同一报告既有 PDF 原件，也有截图版、OCR 版、抽取 JSON。
- 同一批次出现“高质量抽取结果”和“低质量自动结果”。

### 处理原则
- 原始 PDF / 证书原件优先。
- 若多个中间产物指向同一报告编号，只保留字段更完整、来源更可回溯的一版为优先引用，其余标为同源产物。
- 报告类对象不因产品主档已存在就被吞掉；应作为独立证据对象挂接。

## 六、厂家资料去重规则
### 判重主键
- 企业全称
- 企业简称
- 品牌
- 核心产品族
- 原始来源目录

### 处理原则
- 同一家企业只保留一个正式厂家主档。
- 宣传册、报价包、产品册、授权资料挂到同一厂家证据集合。
- 单厂家多型号资料包优先按“厂家主档 → 多型号产品主档 → 报价/配件证据”处理，不平铺成碎页。

## 七、实验批次去重规则
### 当前适用对象
- MinerU 清洗结果
- 报告抽取结果
- 分类测试结果
- 自动命名结果

### 最小记录字段
- 所属批次
- 对应原始文件
- 对象主名称
- 型号
- 制造单位
- 报告编号
- 抽取方式
- 字段完整度
- 是否通过强制字段校验
- 当前结论

### 处理原则
- 实验批次默认不直接升格为正式主档。
- 若同一目录既有分类结果又有报告抽取结果，应拆开记录。
- 自动测试目录默认承担“流程校对”角色，不能因为命名更规范就自动高于原件。

## 八、政策、场景与联系人去重规则
### 判重主键
- 政策：标题、发文单位、发文日期、文号/编号、地区
- 场景：场景名称、地区、行业、来源文件
- 单位联系人：单位名称、地区、联系人姓名、联系人角色、公开联系方式

### 处理原则
- 同一政策的正式文件、解读稿、转载稿不能混成一条事实层记录；正式文件优先，解读稿挂辅助证据。
- 同一场景若只是不同来源对同一公开机会清单的转写，应归并到同一场景对象，不重复建页。
- 单位与联系人默认拆分理解：单位可长期存在，联系人线索可能阶段性变化；联系人信息不足时，优先保留单位级对象。

## 九、当前高频命名差异处理
- `PQ16 / QP16`：统一主名称，变体写入 aliases。
- `高倍数泡沫发生器 / 高倍数泡沫产生器`：统一主名称，原文保留在证据层。
- `刺穿 / 穿刺`：统一主名称，避免拆成两个对象。
- `检验 / 检测 / 检定 / 校准`：先分报告类型，再做去重，不直接混为“报告”。

## 十、保留优先级
1. 正式原件
2. 可回溯结构化抽取
3. 自动实验结果
4. 推断性摘要

## 十一、如何使用这页
1. 新资料进入时，先判断属于哪一层。
2. 再按对象类型套用对应判重主键。
3. 发现重复时，不急着删，先挂回同一对象。
4. 若规则不足，先补本页，再回写模板或流程。
