# 证据链等级说明

TLDR: 这页用于统一所有模板里的证据等级口径，后续不再按页面各自发明“正式资质 / 原厂资料 / 结构参考稿 / 公开辅助信息”等不一致表述，而统一映射到同一套 L1-L5 等级。

## 一、用途
- 统一产品、方案、商务、厂家、政策、单位联系人模板中的证据等级写法。
- 为主档、证据层、实验产物层提供一致的可信度判断口径。
- 让后续去重、归档、入库判断、人工复核优先级能够共用同一套标准。

## 二、统一等级口径
### L1 正式原件
适用：
- 正式合同原件
- 正式证照原件
- 官方正式文件
- 原始 PDF / Word / Excel / 图片原件且来源明确

处理建议：
- 优先作为主证据
- 可直接作为主档挂接核心证据

### L2 官方或原厂公开资料
适用：
- 官网公开资料
- 原厂画册/产品手册
- 官方通知、公示、公开附件
- 官方公开联系人与公开接口信息

处理建议：
- 可作为稳定参考证据
- 适合补主档字段，但仍应保留来源

### L3 人工整理且可回溯
适用：
- 人工清洗 Markdown
- 人工整理摘要
- 人工拆分后的结构稿
- 已能明确回溯到原始文件的内部整理稿

处理建议：
- 可作为工作主引用
- 适合回填主档，但不能替代原件存在

### L4 自动抽取或转写产物
适用：
- OCR 结果
- 自动字段抽取结果
- MinerU / 解析工具生成稿
- 自动命名、自动分类结果

处理建议：
- 默认作为辅助复核层
- 不直接升格为最终事实，除非已人工复核

### L5 宣传材料或待复核
适用：
- 宣传稿
- 行业转载
- 单向营销资料
- 来源模糊的截图、转述、二手整理
- 当前无法确认真伪或边界的线索

处理建议：
- 默认不直接回填主档事实
- 更适合作为线索、候选证据或待复核对象

## 三、旧口径到新口径的映射建议
| 旧表述 | 统一映射 |
| --- | --- |
| 正式证照、正式资质、正式红头文件、正式交付稿 | L1 |
| 原厂资料、官网材料、官方公开稿、公开正式信息 | L2 |
| 内部整理、结构参考稿、内部管理稿、公开辅助信息 | L3 |
| 自动抽取结果、转写结果、解析稿 | L4 |
| 宣传材料、行业转载、待复核、内部线索 | L5 |

## 四、如何使用
1. 每个模板都统一写 L1-L5，不再单独发明等级名称。
2. 若页面需要保留更细的证据类型，用“证据类型”字段表达，不在“证据等级”里混写。
3. 若一个材料同时具备多个来源，以当前实际引用的那份证据为准。
4. 若后续需要更细粒度排序，可在 L1-L5 下再加“高/中/低复核优先级”，但不先扩级。

## 五、与其他页面的关系
- `证据类型` 解决“它是什么材料”
- `证据等级` 解决“它多可信、该放哪一层”
- `日期有效性` 解决“它现在是否仍可直接使用”
- `去重规则` 解决“它是不是和别的其实是同一个对象”
